For the Dispatch App dev:
New: Rental Taxi Requests
When a customer books a rental car and then uses their discount voucher to book a taxi to the pickup depot, a record is written to:

rentalTaxiRequests/{requestId}
  customerName, customerPhone
  pickup (where to pick them up)
  destination (rental company depot)
  scheduledAt (timestamp)
  promoCode, discountPercent
  status: "pending" → you update this
  createdAt

The dispatch app needs to read from this path and assign a driver. The SA portal and Owner Portal can see these requests but currently don't dispatch them automatically — that's on your side.

Update status like this:

PATCH rentalTaxiRequests/{requestId}
  { status: "confirmed" | "completed" | "cancelled" }

For the Passenger App dev:
New: Rental Promo Code Validation
After a customer completes a rental booking, they receive a personal promo code for a discounted taxi ride. To validate it on your side, call:

GET /api/rent/validate-promo?code=R2R-XXXXXX
→ { valid: true/false, discountPercent: 20 }

New: Taxi Booking with Promo
The /ride page on this server handles the full taxi booking form with promo support. If you're building this natively in the passenger app instead, when submitting a taxi ride with a promo code, write directly to:

rentalTaxiRequests/{requestId} (same structure as above)

Then mark the promo as used:

PATCH rentalPromos/{code} → { status: "used", usedAt: timestamp }

New: Rental Booking Lookup
Customers can look up their rental booking at /rent/booking using their email + booking reference. No action needed from passenger app unless you want to deep-link into it.

For the Website dev (public bookawaka.co.nz site):
Company Registration Endpoint:

POST /api/public/register
Content-Type: application/json
{
  companyName, contactName, email, phone,
  city, country,
  businessType: "taxi" | "food" | "courier" (comma-separated for multiple),
  fleetSize, website, message,
  tradingName (optional), regNo (optional)
}
→ { ok: true, ref: "REQ17xxxxxxxxx" }

Submissions are stored in Firebase at onboardRequests/{ref} and appear in the SA portal under Onboarding Requests automatically.
