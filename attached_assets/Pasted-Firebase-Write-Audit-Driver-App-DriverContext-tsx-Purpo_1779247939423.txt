Firebase Write Audit — Driver App (DriverContext.tsx)
Purpose: Categorize every direct Firebase write call in the driver app before the 22c cutover, so the server dev can flag anything ambiguous before we remove it. File: context/DriverContext.tsx.

Categories:

KEEP-presence — driver location / vehiclestatus / OnTheWay/Arrived/MeterOn timestamps. Stays as direct Firebase write per spec.
KEEP-chat — driverMsg/ + messages/. Stays.
KEEP-profile — driver identity / push tokens / vehicle profile / shift logs / ratings. Stays.
KEEP-cleanup — onDisconnect handlers and presence teardown.
REPLACE — booking state write. Becomes a POST /api/job/command call.
DELETE — write that becomes redundant because server owns the state (no replacement needed).
🚩 SERVER-DEV-EYES — borderline cases the server dev asked to review.
